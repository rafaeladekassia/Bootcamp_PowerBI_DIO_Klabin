
# Hands-on Project: Repair Shop Database
## Project Description

The project aims to create from scratch a database for an Repair Shop to manage **clients, service orders, vehicles, parts stock, services offered, and employees**, as well as data analysis for business insights.  The database was created according to the following steps:

* writing the SQL script for al the tables included;
* inserting persisted data;
* creating queries to solve specific business questions.

> *All data used here is fictional and for educational purpose.*

## Business Questions

The following business questions were considered in this project:
* What was the most complex order (more services)?
* How much does it cost the most expensive order?

## Data modelling
The script makes use of the following main entities:

* **Clients:** Register clients through an unique identifier (Tax ID).
* **Service order:** Register services requested by clients for one or more vehicles.
* **Vehicles:** Store information on vehicles, with an unique identifier (plate).
* **Parts stock:** Manage availability of parts for use in repair services.
* **Services offered:** List services provided (repair of cars, motorcycles, and  bike) and related information (price).
* **Employees:** Register individual employees on the database and services (Mechanic, Attendant, Manager, Cashier).
